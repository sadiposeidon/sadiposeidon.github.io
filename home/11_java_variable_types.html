Java - Değişken Türleri

Java Değişkeni nedir?

Bir değişken bize programlarımızın işleyebileceği adlandırılmış depolama alanı sağlar. Java'daki her değişkenin, değişkenin belleğinin boyutunu ve düzenini belirleyen belirli bir türü vardır. Söz konusu hafızada saklanabilecek değerlerin aralığı ve değişkene uygulanabilecek işlemler kümesi.

Değişken Bildirimi ve Başlatma (Variable Declaration and Initialization)

Kullanılmadan önce tüm değişkenleri bildirmeniz gerekir. Değişken bildiriminin temel biçimi aşağıdadır.
data type variable [ = value][, variable [ = value] ...] ;
Burada veri türü , Java'nın veri türlerinden biridir ve değişken, değişkenin adıdır. Belirtilen türde birden fazla değişken bildirmek için virgülle ayrılmış bir liste kullanabilirsiniz.

Geçerli Değişken Bildirimleri ve Başlatma Örnekleri

Aşağıda Java'da değişken bildirimi ve başlatmanın geçerli örnekleri verilmiştir. 
int a, b, c;            // a, b ve c olmak üzere üç int'i bildirir.
int a = 10, b = 10;     // Başlatma örneği
byte B = 22;            // bayt tipi değişken B'yi başlatır.
double pi = 3.14159;    // PI değerini bildirir ve atar.
char a = 'a';           // a i karakter değişkeni 'a' değeriyle başlatıldı

Java Değişken Türleri

Aşağıda üç tür Java değişkeni yer almaktadır.

Yerel değişkenler (Local variables) Örnek değişkenler (Instance variables) Sınıf / Statik değişkenler (Class / Static variables)

1. Java Yerel Değişkenleri (Local Variables)

Yerel değişkenler yöntemlerde (methods), yapıcılarda (constructors) veya bloklarda (blocks) bildirilir.

Yerel değişkenler yönteme, yapıcıya veya bloğa girildiğinde oluşturulur ve değişken yöntemden, yapıcıdan veya bloktan çıktığında yok edilir.

Erişim değiştiricileri (Access modifiers) yerel değişkenler için kullanılamaz.

Yerel değişkenler yalnızca bildirilen yöntem (declared method), yapıcı (constructor) veya blok (block) içinde görünür.

Yerel değişkenler dahili olarak yığın (stack) düzeyinde uygulanır.

Yerel değişkenler için varsayılan bir değer yoktur, bu nedenle ilk kullanımdan önce yerel değişkenler bildirilmeli ve bir başlangıç değeri atanmalıdır.

Örnek 1: Değişkenin başlatmayla birlikte yerel kapsamı

Burada yaş yerel bir değişkendir. Bu, pupAge() yönteminin içinde tanımlanır ve kapsamı yalnızca bu yöntemle sınırlıdır. 

public class Test {
   public void pupAge() {

      int age = 0;
      age = age + 7;

      System.out.println("Puppy age is : " + age);
   }

   public static void main(String args[]) {

      Test test = new Test();
      test.pupAge();
   }
}

Örnek 2: Değişkenin başlatma olmadan yerel kapsamı

Aşağıdaki örnek, onu başlatmadan age kullanıyor, bu nedenle derleme sırasında hata verecektir. 

public class Test {
   public void pupAge() {
      int age;
      age = age + 7;
      System.out.println("Puppy age is : " + age);
   }

   public static void main(String args[]) {
      Test test = new Test();
      test.pupAge();
   }
}

2. Java Örnek Değişkenler (Instance Variables)

Örnek değişkenler bir sınıfta ancak bir yöntemin, yapıcının veya herhangi bir bloğun dışında bildirilir .

Heap'teki bir nesne için bir alan tahsis edildiğinde, her örnek değişken değeri için bir yuva oluşturulur.

Örnek değişkenler, 'yeni' anahtar kelimesi kullanılarak bir nesne oluşturulduğunda oluşturulur ve nesne yok edildiğinde yok edilir.

Örnek değişkenler, birden fazla yöntem, kurucu veya blok tarafından başvurulması gereken değerleri veya sınıf boyunca mevcut olması gereken bir nesnenin durumunun önemli kısımlarını tutar.

Örnek değişkenler kullanımdan önce veya sonra sınıf düzeyinde bildirilebilir.

Örnek değişkenler için erişim değiştiricileri verilebilir.

Örnek değişkenler sınıftaki tüm yöntemler, yapıcılar ve blok için görülebilir. Normalde bu değişkenlerin özel (erişim düzeyi) yapılması önerilir. Ancak erişim değiştiricilerin kullanımıyla bu değişkenler için alt sınıfların görünürlüğü sağlanabilir.

Örnek değişkenlerin varsayılan değerleri vardır. Sayılar için varsayılan değer 0'dır, Booleans için false'tur ve nesne referansları için null'dur. Değerler bildirim sırasında veya yapıcı içinde atanabilir.

Örnek değişkenlere, sınıf içindeki değişken adı çağrılarak doğrudan erişilebilir. Ancak statik yöntemler içerisinde (örnek değişkenlerine erişilebilirlik verildiğinde), bunların tam nitelenmiş ad kullanılarak çağrılması gerekir. ObjectReference.VariableName

Java Örnek Değişkenleri Örneği


import java.io.*;

public class Employee {

   // bu örnek değişken herhangi bir alt sınıf için görülebilir.
   public String name;

   // maaş değişkeni yalnızca Çalışan sınıfında görünür.
   private double salary;

   // Yapıcıda isim değişkeni atanır.
   public Employee (String empName) {

      name = empName;
   }

   // Maaş değişkenine bir değer atanır.
   public void setSalary(double empSal) {

      salary = empSal;
   }

   // Bu yöntem çalışan ayrıntılarını yazdırır.
   public void printEmp() {

      System.out.println("name  : " + name );
      System.out.println("salary :" + salary);
   }

   public static void main(String args[]) {

      Employee empOne = new Employee("Ransika");
      empOne.setSalary(1000);
      empOne.printEmp();
   }
}

3. Java Sınıf / Statik Değişkenler (Class / Static Variables)

Statik değişkenler olarak da bilinen sınıf değişkenleri, bir sınıfta statik anahtar kelimeyle ancak bir yöntemin, yapıcının veya bloğun dışında bildirilir.

Ondan kaç nesne oluşturulduğuna bakılmaksızın, her sınıf değişkeninin sınıf başına yalnızca bir kopyası olacaktır.

Statik değişkenler, sabit olarak bildirilmek dışında nadiren kullanılır. Sabitler, genel/özel, final ve statik olarak bildirilen değişkenlerdir. Sabit değişkenler hiçbir zaman başlangıç değerlerinden değişmezler.

Statik değişkenler statik bellekte saklanır. Bildirilen final dışında statik değişkenlerin kullanılması ve genel veya özel sabitler olarak kullanılması nadirdir.

Statik değişkenler program başlatıldığında oluşturulur ve program durdurulduğunda yok edilir.

Görünürlük örnek değişkenlere benzer. Ancak çoğu statik değişken, sınıfın kullanıcıları tarafından kullanılabilir olması gerektiğinden, genel olarak bildirilir.

Varsayılan değerler örnek değişkenlerle aynıdır. Sayılar için varsayılan değer 0'dır; Booleans için bu yanlıştır; ve nesne referansları için null'dur. Değerler bildirim sırasında veya yapıcı içinde atanabilir. Ek olarak, özel statik başlatıcı bloklarında değerler atanabilir.

Statik değişkenlere ClassName.VariableName sınıf adı ile çağrılarak erişilebilir .

Sınıf değişkenleri genel statik final olarak bildirildiğinde, değişken adlarının (sabitlerin) tümü büyük harfle yazılır. Statik değişkenler genel ve nihai değilse, adlandırma sözdizimi örnek ve yerel değişkenlerle aynıdır.

Java Sınıfı/Statik Değişkenleri Örneği


import java.io.*;

public class Employee {

   // salary  variable is a private static variable
   private static double salary;

   // DEPARTMENT is a constant
   public static final String DEPARTMENT = "Development ";

   public static void main(String args[]) {
      salary = 1000;
      System.out.println(DEPARTMENT + "average salary:" + salary);
   }
}

Not − Değişkenlere dışarıdan bir sınıftan erişiliyorsa sabite Employee.DEPARTMENT olarak erişilmelidir.
