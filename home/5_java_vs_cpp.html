Java ve C++

Java genel amaçlı, üst düzey bir programlama dilidir. Java, web geliştirme, Makine Öğrenimi ve diğer ileri düzey yazılım geliştirme için kullanılır. Java programlama dili ilk olarak James Gosling tarafından başlatılan ve Sun Microsystems'in Java platformunun (Java 1.0 [J2SE]) temel bileşeni olarak 1995 yılında piyasaya sürülen Sun Microsystems tarafından geliştirildi.

C++ orta düzey, büyük/küçük harfe duyarlı, nesne yönelimli bir programlama dilidir. Bjarne Stroustrup, Bell Laboratuvarlarında C++'ı yarattı. C++, Windows, Mac OS ve Linux'ta çalışan, platformdan bağımsız bir programlama dilidir. C++ donanıma yakındır ve düşük seviyeli programlamaya izin verir. Bu, geliştiriciye bellek üzerinde kontrol, gelişmiş performans ve güvenilir yazılım sağlar.

C++ ve Java'ya genel bir bakış ve bu iki programlama dilinin birbirinden nasıl farklı olduğunu öğrenmek için bu makaleyi okuyun.

Java nedir?

Java Standard Edition'ın en son sürümü Java SE 21'dir. Java'nın ilerlemesi ve yaygınlaşmasıyla birlikte, çeşitli platform türlerine uyacak şekilde birden fazla yapılandırma oluşturuldu. Örneğin: Kurumsal Uygulamalar için J2EE, Mobil Uygulamalar için J2ME.

Yeni J2 sürümleri sırasıyla Java SE, Java EE ve Java ME olarak yeniden adlandırıldı. Java'nın Bir Kez Yaz, Her Yerde Çalıştır özelliği garanti edilir.

Java'nın özellikleri

Nesneye yönelik (Object Oriented): Java'da her şey bir Object'tir. Java, Object modelini temel aldığından kolaylıkla genişletilebilir.

Platformdan bağımsız (Platform Independent): C ve C++ da dahil olmak üzere diğer birçok programlama dilinden farklı olarak Java derlendiğinde, platforma özgü makinede değil, platformdan bağımsız bayt kodunda derlenir. Bu bayt kodu web üzerinden dağıtılır ve hangi platformda çalıştırılırsa çalıştırılsın Java Sanal Makinesi (JVM) tarafından yorumlanır.

Basit (Simple): Java öğrenmesi kolay olacak şekilde tasarlanmıştır. OOP Java'nın temel konseptini anlarsanız , ustalaşması kolay olacaktır.

Güvenli (Secure): Java'nın güvenli özelliği sayesinde virüssüz, kurcalanmaya karşı korumalı sistemler geliştirmenize olanak sağlar. Kimlik doğrulama teknikleri genel anahtar şifrelemesine dayanır.

Mimariden bağımsız (Architecture-neutral): Java derleyicisi, Java çalışma zamanı sisteminin varlığıyla derlenmiş kodu birçok işlemcide yürütülebilir hale getiren, mimariden bağımsız bir nesne dosyası formatı oluşturur.

Taşınabilir (Portable): Mimariden bağımsız olması ve spesifikasyonun uygulamaya bağımlı yönlerinin bulunmaması, Java'yı taşınabilir kılar. Java'daki derleyici, bir POSIX alt kümesi olan temiz taşınabilirlik sınırıyla ANSI C'de yazılmıştır.

Sağlam (Robust): Java, esas olarak derleme zamanı hata denetimi ve çalışma zamanı denetimi üzerinde durarak hataya açık durumları ortadan kaldırmak için çaba gösterir.

Çok iş parçacıklı (Multithreaded): Java'nın çok iş parçacıklı özelliği sayesinde aynı anda birçok görevi gerçekleştirebilen programlar yazmak mümkündür. Bu tasarım özelliği, geliştiricilerin sorunsuz çalışabilen etkileşimli uygulamalar oluşturmasına olanak tanır.

Yorumlanmış (Interpreted): Java bayt kodu anında yerel makine talimatlarına çevrilir ve hiçbir yerde saklanmaz. Bağlantının artımlı ve hafif bir süreç olması nedeniyle geliştirme süreci daha hızlı ve analitiktir.

Yüksek performans (High Performance): Tam Zamanında derleyicilerin kullanımıyla Java, yüksek performans sağlar.

Dağıtılmış (Distributed): Java, internetin dağıtılmış ortamı için tasarlanmıştır.

Dinamik (Dynamic): Java , gelişen ortama uyum sağlayacak şekilde tasarlandığından, C veya C++' dan daha dinamik olarak kabul edilir . Java programları, çalışma zamanındaki nesnelere erişimleri doğrulamak ve çözümlemek için kullanılabilecek geniş miktarda çalışma zamanı bilgisi taşıyabilir.

Java Örneği

Aşağıdaki basit Java programına bir göz atın. 

package com.example;

import java.util.Scanner;

public class JavaTester {

   public static void main(String args[]) {

      String a, b;
      Scanner scanner = new Scanner(System.in);  

      System.out.println("a değişkeninin değerini girin");
      a = scanner.nextLine();

      System.out.println("b değişkeninin değerini girin");
      b = scanner.nextLine();

      System.out.println("a için girdiğiniz değer: " + a);
      System.out.println("b için girdiğiniz değer: " + b);

      scanner.close();
   }       
}
Örneğimizde "a" ve "b" adlı iki değişkeni aldık ve bu değişkenlere bir miktar değer atadık. Java'da, değişkenler için veri türünü açıkça bildirmemiz gerektiğini unutmayın, çünkü Java kesin olarak yazılan bir dildir. Java nesne yönelimli bir dil olduğundan, herhangi bir eylemi gerçekleştirmek için nesneleri kullanırız. Örnekte, System.in nesnesi tarafından temsil edilen konsoldan kullanıcı girişini okumak için Scanner sınıfı nesnesini kullandık. Alınan değerleri yazdırmak için System.out nesne yöntemi println() kullanılır.

C++ nedir?

C++, dik bir öğrenme eğrisine sahip, statik olarak yazılmış, derlenmiş, çok paradigmalı, genel amaçlı bir programlama dilidir. Video oyunları, masaüstü uygulamaları ve gömülü sistemler onu yaygın olarak kullanıyor. C++, C ile o kadar uyumludur ki neredeyse tüm C kaynak kodlarını hiçbir değişiklik yapmadan oluşturabilir. Nesne yönelimli programlama, C++'ı C'den daha iyi yapılandırılmış ve daha güvenli bir dil haline getirir. C++'ın özellikleri

C++'ın bazı özelliklerine ve popülerliğinin nedenine bakalım.

Orta düzey dil (Middle-level language): Hem sistem geliştirme hem de Medya Oynatıcılar, Photoshop, Oyun Motorları vb. gibi büyük ölçekli tüketici uygulamaları için kullanılabildiğinden orta düzey bir dildir.

Yürütme hızı (Execution Speed): C++ kodu hızlı çalışır. Çünkü derlenmiş ve prosedürleri yoğun bir şekilde kullanıyor. Çöp toplama, dinamik yazma ve diğer modern özellikler programın yürütülmesini engeller.

Nesne yönelimli dil (Object-oriented language): Nesne yönelimli programlama esnek ve yönetilebilirdir. Büyük uygulamalar mümkündür. Kodun büyümesi prosedür kodunun işlenmesini zorlaştırır. C++'ın C'ye göre en önemli avantajı.

Kapsamlı kütüphane desteği (Extensive Library Support): C++ geniş bir kütüphaneye sahiptir. Hızlı geliştirme için üçüncü taraf kütüphaneler desteklenir.

C++ Örneği

Aşağıda yazılı bir örnek üzerinden C++'ın sözdizimini anlayalım. 

#include

int main() {

Örneğimizde "a" ve "b" adlı iki değişkenin klavye aracılığıyla kullanıcıdan girdisini alıyoruz ve verileri konsolda görüntülüyoruz.

Java ve C++ Arasındaki Fark

Hem Java hem de C++ en popüler programlama dilleri arasındadır. Her ikisinin de avantajları ve dezavantajları var. Bu dersimizde onları birbirinden ayıran karakteristik özelliklere yakından bakacağız.

1. Tarafından geliştirilmiş (Developed by): Java, Sun Microsystems'da James Gosling tarafından geliştirildi. Başlangıçta gömülü sistemler, set üstü kutular, televizyonlar vb. için tasarlandı. Daha sonra internet tabanlı uygulama geliştirmede tercih edilen dil haline geldi

1. Tarafından geliştirilmiş (Developed by): C++, Bell Laboratuvarlarında Bjarne Stroustrup tarafından C dilinin bir uzantısı olarak geliştirildi. Hem yüksek seviyeli hem de düşük seviyeli makine kodu erişimini destekler. C++ esas olarak sistem yazılımları, derleyiciler vb. geliştirmek için kullanılır.

2. Tarafından etkilenmiş (Influenced by): Ada 83, Pascal, C++, C#'dan etkilenmiştir.

2. Tarafından etkilenmiş (Influenced by): Ada, ALGOL 68, C, ML, Simula, Smalltalk'tan etkilendi.

3. Mimariye bağımlılık (Dependency on Architecture): Java bayt kodu herhangi bir işletim sisteminde çalışır. Bayt kodu JVM için hedeflenmiştir. JVM veya Java Sanal Makinesi daha sonra bayt kodunu yorumlar ve temeldeki makineye özel kodu çalıştırır. Bu nedenle Java kodunun farklı makinelerde çalışacak şekilde değiştirilmesine gerek yoktur.

3. Mimariye bağımlılık (Dependency on Architecture): Kütüphaneler farklı sistemlerde farklı olduğundan her işletim sisteminde çalışmaz.

4. Platform bağımsızlığı (Platform independence): Herhangi bir işletim sistemi üzerinde çalışabilir. Java kodu platformdan bağımsızdır. Platforma özel kod gerekmez. int, long'un boyutu tüm platformlarda aynı kalır.

4. Platform bağımsızlığı (Platform independence): Farklı platformlarda farklı şekilde derlenir, herhangi bir işletim sisteminde çalıştırılamaz.

5. Taşınabilirlik (Portablity): Taşınabilir. Platformdan bağımsız olduğundan, bir java kodu herhangi bir platforma özel değişiklik yapılmadan herhangi bir makinede olduğu gibi aktarılabilir. Windows makinesinde yazılan bir java kodu, Unix makinesinde hiçbir değişiklik yapılmadan aynı şekilde çalışabilir.

5. Taşınabilirlik (Portablity): Taşınabilir değildir.

6. Yorumlanmış / Derlenmiş (Interpreted/Compiled): Yorumlanan bir dildir.

6. Yorumlanmış / Derlenmiş (Interpreted/Compiled): Derlenmiş bir dildir.

7. Bellek yönetimi (Memory management): Bellek yönetimi otomatik olarak yapılır. Java, bir nesne gerekmediğinde belleği otomatik olarak serbest bırakan Çöp Toplayıcı (Garbage Collector) hizmetini sağlar.

7. Bellek yönetimi (Memory management): Bellek yönetimi manuel olarak yapılmalıdır.

8. Sanal anahtar (virtual Keyword): 'Sanal' anahtar kelimesi yok.

8. Sanal anahtar (virtual Keyword): 'Sanal' anahtar kelimesine sahiptir.

9. Çoklu kalıtım desteği (Multiple Inheritance support): Yalnızca tekli mirası destekler. Arayüzler (yalnızca kısmi) kullanılarak çoklu kalıtım elde edilebilir. Bir sınıf yalnızca tek bir sınıfı genişletebilir. Her ne kadar arayüz çoklu kalıtımı genişletebilse de. Çoklu kalıtım belirsiz sonuçlara yol açabilir. Virtual anahtar sözcüğü Java'da desteklenmediğinden çoklu miras desteklenmez.

9. Çoklu kalıtım desteği (Multiple Inheritance support): Tekli ve çoklu Kalıtımı destekler. Sanal anahtar kelime kullanılarak belirsiz referans çözülebilir.

10. Operatör aşırı yükleme desteği (operator overloading support): Operatörün aşırı yüklenmesini desteklemez. Java yalnızca yöntemin aşırı yüklenmesini destekler. Operatör aşırı yüklemesinin temel dile karmaşıklık kattığı düşünülür ve bu nedenle dili basit tutmak için uygulanmaz.

10. Operatör aşırı yükleme desteği (operator overloading support): Operatörün aşırı yüklenmesini destekler. C++'da hem yöntemleri hem de operatörleri aşırı yükleyebiliriz.

11. İşaretçiler (pointers): İşaretçilere sınırlı destek sağlar. İşaretçiler karmaşık bir işlevsellik olduğundan Java bunları kullanmaktan kaçınır. Nesnelere veya tam olarak onların adreslerine işaret etmek için referans kavramını sağlar.

11. İşaretçiler (pointers): İşaretçi işlemlerini destekler. Geliştiriciler karmaşık işlemleri gerçekleştirebilir, işaretçileri kullanarak optimize edilmiş bellek tabanlı kod yazabilir. Ancak oldukça karmaşıktır ve bu konularda uzmanlaşmak güçlü programlama becerileri gerektirir.

12. Düşük seviyeli makine kodu erişimi (Low level machine code access): Üst düzey işlevlere sahiptirler. Java, platformdan bağımsız bir dildir ve Java'nın bayt kodu olarak derlenmiş kodu, kodu daha da düşük seviyeli koda dönüştüren JVM içindir. Dolayısıyla geliştiriciler Java kullanarak düşük seviyeli makine kodu yazamazlar. Java'nın esas olarak uygulama geliştirmede kullanılmasının nedeni budur.

12. Düşük seviyeli makine kodu erişimi (Low level machine code access): Düşük seviyeli işlevlere sahiptirler. C++ düşük seviyeli makine kodu kodunu desteklediğinden. Esas olarak sistem yazılımları, derleyiciler vb. yazmak için kullanılır.

13. Yerel kütüphanelere erişim (Native libraries access): Doğrudan yerel kütüphane çağrılarını desteklemez. Java, düşük seviyeli makine koduyla çalışacak şekilde tasarlanmamıştır ve yerel çağrıları desteklemez. Ancak yerel çağrıları üçüncü taraf kütüphaneleri kullanarak yapılandırabiliriz.

13. Yerel kütüphanelere erişim (Native libraries access): Doğrudan sistem kitaplığı çağrılarını destekler.

14. Dokümantasyon yorumu (documentation comment): Kaynak kodu için belge yorumunu (/**.. */) destekler . Javadoc aracı, kaynak koddan dokümantasyon yorumlarını okuyabilir ve yorumlara dayalı olarak html tabanlı Java dokümantasyonu oluşturabilir.

14. Dokümantasyon yorumu (documentation comment): Kaynak kodu için dokümantasyon yorumunu desteklemez.

15. Çoklu iş parçacığı (Multithreading): İş parçacığı işlemlerini destekler. Java varsayılan olarak çoklu iş parçacığı desteğine sahiptir. Eşzamanlı programlamanın verimliliği artırmasına ve harcanan zamanı azaltmasına olanak tanır

15. Çoklu iş parçacığı (Multithreading): Tasarım gereği konuları desteklemiyor. Üçüncü taraf iş parçacığı kitaplıkları kullanılarak yapılabilir.

16. Konsol girişi (Console Input): Giriş için 'System' sınıfını, yani System.in'i kullanır. System.in sınıfı, konsoldaki kullanıcıdan girdi almak için kullanılabilir.

16. Konsol girişi (Console Input): Giriş işlemi için 'cin'i kullanır. cin kullanıcının konsola değer girmesine izin verir.

17. Konsol çıkışı (Console Output): Çıkış için System.out'u kullanır. System.out.println() yöntemi, gerekli değeri sistem konsoluna yazdırır.

17. Konsol çıkışı (Console Output): Çıkış işlemi için 'cout'u kullanır. cout gerekli değeri sistem konsoluna yazdırır.

18. Küresel destek (global support): Küresel kapsamı desteklemiyor. Java katı bir nesne yönelimli dildir ve genel kapsam mevcut değildir. Paketleri kullanarak paket kapsamını destekler.

18. Küresel destek (global support): Ad alanı kapsamının yanı sıra küresel kapsamı da destekler.

19. Yapı / sendika desteği (struct/union support): Yapıları ve birlikleri desteklemez.

19. Yapı / sendika desteği (struct/union support): Yapıları ve sendikaları destekler.

20. Git anahtarı (goto keyword): 'Goto' anahtar kelimesine sahip değil. Ancak aynı işlevselliğe etiketler kullanılarak da ulaşılabilir. Bir break/continue ifadesi etiketli bir ifade konumuna atlayabilir.

20. Git anahtarı (goto keyword): 'Goto' anahtar kelimesini destekler. Goto anahtar sözcüğünü kullanarak etiketli herhangi bir konuma atlayabiliriz.

21. Değere / referansa göre geçiş (pass by value/reference): Yalnızca değere göre geçiş yöntemini destekler. Nesne referansı bile teknik olarak bir yönteme değer olarak aktarılır.

21. Değere / referansa göre geçiş (pass by value/reference): Değere göre geçiş ve referans yöntemlerine göre geçiş'i destekler. Referansla geçiş durumunda işaretçi veya & gösterimi gereklidir.

22. Bellek yönetimi (Memory Management): Çöp toplayıcıyı kullanarak nesne yönetimini otomatik olarak gerçekleştirir. Geliştiricilerin nesneler için bellek ayırması veya nesneler kullanılmadığında bunların yerini değiştirmesi gerekmez. Çöp Toplayıcı hizmeti otomatik olarak alanı algılar ve boşaltır. GC hizmeti nedeniyle Java'da bellek sızıntısı olasılığı çok daha azdır.

22. Bellek yönetimi (Memory Management): 'Yeni' ve 'sil' yardımıyla nesne yönetimini manuel olarak gerçekleştirir. Geliştiricilerin, bellek sızıntılarını önlemek amacıyla belleğin uygun şekilde tahsis edildiğinden/tahsis edildiğinden emin olmak için önlemler alması gerekir.
