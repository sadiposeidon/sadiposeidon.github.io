Java - Özellikler

Java programlama dili başlangıçta gömülü sistemler (embedded systems), set üstü kutular ve televizyon üzerinde çalışmak üzere geliştirildi . Dolayısıyla gereksinimler gereği başlangıçta çeşitli platformlarda çalışacak şekilde tasarlandı. Birkaç yıl içinde Java, internet tabanlı uygulamalar geliştirmek için kullanılan en popüler dillerden biri haline geldi.

Java zengin özelliklere sahip bir dildir ve her yeni sürümle birlikte sürekli olarak gelişmektedir. Milyarlarca cihazda yaygın olarak kullanılmaktadır. Java dilinin temel özellikleri şunlardır.

Nesne odaklı (Object Oriented)

Java'da her şey bir Nesnedir. Java, Object modelini temel aldığından kolaylıkla genişletilebilir. Nesneye Yönelik özelliğe sahip bir dil olarak Java, OOP'lerin aşağıdaki temel kavramlarını destekler.

Polymorphism (Polimorfizm)

Inheritance (Miras)

Encapsulation (Kapsülleme)

Abstraction (Soyutlama)

Classes (Sınıflar)

Objects (Nesneler)

Instance (Misal)

Method (Yöntem)

Message Passing (İleti geçişi)

Platform bağımsız (Platform Independent)

C ve C++ da dahil olmak üzere diğer birçok programlama dilinin aksine , Java derlendiğinde platforma özel bir makineye değil, platformdan bağımsız bayt koduna derlenir. Bu bayt kodu web üzerinden dağıtılır ve hangi platformda çalıştırılırsa çalıştırılsın Java Sanal Makinesi (JVM) tarafından yorumlanır.

Java Bir Kez Yaz, Her Yerde Çalıştır (Write Once, Run Anywhere) (WORA) yöntemiyle tasarlanmıştır. Java'da yazılan kod, çalıştırdığı makinenin türüne doğrudan bağlı değildir. Bir kod, Java'nın platformdan bağımsız ByteCode'da derlenmesidir. Java Sanal Makinesi, JVM bayt kodunu anlayabilir. Java, platforma özel JVM'ler sağlar. Bayt kodunu doğru yorumlamak platforma özel JVM'nin sorumluluğundadır, böylece geliştiriciler Windows, Linux, Unix, Mac vb. platformlar hakkında endişelenmeden kod yazmakta özgürdür. Bu özellik Java'yı platformdan bağımsız bir dil haline getirir.

Bayt kodu web üzerinden dağıtılabilir ve hangi platformda çalıştırılırsa çalıştırılsın Sanal Makine (JVM) tarafından yorumlanabilir. Java kodunu son derece taşınabilir hale getirir ve birden fazla platformda çalışan uygulamalar için kullanışlıdır.

Basit (Simple)

Java öğrenilmesi kolay olacak şekilde tasarlanmıştır. OOP Java'nın temel konseptini anlarsanız, ustalaşması kolay olacaktır.

Java'yı öğrenmek çok kolaydır. Birçok özelliği C ve C++'dan devralır ve işaretçiler, operatör aşırı yüklemesi, çoklu miras, açık bellek tahsisi vb. gibi karmaşık özellikleri kaldırır. Otomatik çöp toplama olanağı sağlar. Binlerce kullanışlı fonksiyona sahip zengin kütüphane seti ile Java, geliştiricilerin hayatını kolaylaştırır.

Güvenli (Secure)

Java'nın güvenli özelliği sayesinde virüssüz, müdahalesiz sistemler geliştirmenize olanak sağlar. Kimlik doğrulama teknikleri genel anahtar şifrelemesine dayanır.

Java, geliştiricilerin temeldeki sistem belleği veya işletim sistemiyle etkileşim kurmasını istemediği için tasarımı gereği oldukça güvenlidir. Bayt kodu güvenlidir ve arabellek taşması, bellek sızıntısı gibi çeşitli güvenlik kusurları çok nadirdir. Java istisna işleme mekanizması, geliştiricilerin programın yürütülmesi sırasında meydana gelebilecek hemen hemen tüm hata türlerini/istisnaları ele almasına olanak tanır. Otomatik çöp toplama, sistem bellek alanı kullanımının kontrol altında tutulmasına yardımcı olur.

Mimariden bağımsız (Architecture-neutral)

Java derleyicisi (compiler), Java çalışma zamanı sisteminin varlığıyla derlenmiş kodu birçok işlemcide çalıştırılabilir hale getiren, mimariden bağımsız bir nesne dosyası formatı oluşturur.

Java derleyicisi, Java çalışma zamanı sisteminin varlığıyla derlenmiş kodu birçok işlemcide çalıştırılabilir hale getiren, mimariden bağımsız bir nesne dosyası formatı oluşturur. İşlemci mimarilerindeki veya makineye özel işlemcilerdeki ilerlemelerle birlikte Java kodu, işlemcinin herhangi bir özel gereksiniminden bağımsız kalır. Java açık bir standart olduğundan, özel bir mimariye özel bir JVM bile hazırlanabilir. Günümüzde olduğu gibi, hemen hemen tüm popüler platformlar, mimariler için JVM'miz mevcut olup, java kodu tamamen bağımsızdır. Örneğin Windows makinede oluşturulan bir java programı, Linux makinede herhangi bir kod değişikliği yapılmadan çalışabilir.

Taşınabilir (Portable)

Mimariden bağımsız olması ve spesifikasyonun uygulamaya bağımlı yönlerinin bulunmaması Java'yı taşınabilir kılar. Java'daki derleyici, bir POSIX alt kümesi olan temiz taşınabilirlik sınırıyla ANSI C'de yazılmıştır.

Bu taşınabilirlik nedeniyle Java, başlangıcından bu yana anında popüler oldu. Platformların yerden yere değiştiği ve aynı kod tabanının birden fazla platformda kullanılabildiği internet tabanlı uygulamalar için özellikle yararlıydı. Dolayısıyla birden fazla lokasyonda geliştiriciler arasındaki işbirliği kolaydı.

Güçlü (Robust)

Java, esas olarak derleme zamanı hata denetimi ve çalışma zamanı denetimi üzerinde durarak hataya açık durumları ortadan kaldırmak için çaba gösterir. Otomatik çöp toplama, güçlü bellek yönetimi, işaretçi yok, sistem belleğine doğrudan erişim yok, istisna yönetimi, hata işleme, Java'yı güvenilecek sağlam ve güçlü bir dil yapan temel özelliklerden bazılarıdır.

Çok iş parçacıklı (Multithreaded)

Java'nın çok iş parçacığı özelliği sayesinde birçok görevi aynı anda gerçekleştirebilen programlar yazmak mümkündür. Bu tasarım özelliği, geliştiricilerin sorunsuz çalışabilen etkileşimli uygulamalar oluşturmasına olanak tanır.

Çok iş parçacıklı bir program, eşzamanlı olarak çalışabilen iki veya daha fazla parça içerir ve her parça, özellikle bilgisayarınızda birden fazla CPU varsa, mevcut kaynaklardan en iyi şekilde yararlanarak aynı anda farklı bir görevi yerine getirebilir.

Tanım gereği çoklu görev, birden fazla işlemin CPU gibi ortak işlem kaynaklarını paylaşmasıdır. Çoklu iş parçacığı , çoklu görev fikrini, tek bir uygulama içindeki belirli işlemleri ayrı iş parçacıklarına bölebileceğiniz uygulamalara genişletir. Her bir iş parçacığı paralel olarak çalışabilir. İşletim sistemi, işlem süresini yalnızca farklı uygulamalar arasında değil, aynı zamanda bir uygulama içindeki her bir iş parçacığı arasında da bölüştürür.

Çoklu iş parçacığı, aynı programda birden fazla etkinliğin eş zamanlı olarak ilerleyebileceği şekilde yazmanıza olanak tanır.

Yorumlandı (Interpreted)

Java bayt kodu anında yerel makine talimatlarına çevrilir ve hiçbir yerde saklanmaz. Bağlantının artımlı ve hafif bir süreç olması nedeniyle geliştirme süreci daha hızlı ve analitiktir.

JVM, javac derleyicisi ile temeldeki donanım arasında yer alır; javac (veya başka herhangi bir derleyici) derleyicisi, platforma özgü bir JVM tarafından anlaşılan Bytecode'da Java kodunu derler. JVM daha sonra kod yürütülürken JIT (Tam zamanında) derlemesini kullanarak Bayt kodunu ikili olarak derler.

Yüksek performans (High Performance)

Java, Just-In-Time derleyicilerinin kullanımıyla yüksek performans sağlar. JVM, programın yürütme süresini iyileştirmek için JIT derleyicisini kullanır. Aşağıda JIT derleyicileri tarafından yapılan bazı genel optimizasyonlar verilmiştir.

Yöntem satır içi (Method inlining)

Ölü kodun ortadan kaldırılması (Dead code elimination)

Çağrı sitelerini optimize etmek için buluşsal yöntemler (Heuristics for optimizing call sites)

Sürekli katlama (Constant folding)

Dağıtılmış (Distributed)

Java, internetin dağıtılmış ortamı için tasarlanmıştır.

Dinamik (Dynamic)

Java, gelişen ortama uyum sağlayacak şekilde tasarlandığından, C veya C++'dan daha dinamik olarak kabul edilir. Java programları, çalışma zamanındaki nesnelere erişimleri doğrulamak ve çözümlemek için kullanılabilecek geniş miktarda çalışma zamanı bilgisi taşıyabilir.
