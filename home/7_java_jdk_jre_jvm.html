Java - JDK'ya karşı JRE'ye karşı JVM

JDK, JRE ve JVM'nin üçü de birbirine bağımlıdır. JDK, öncelikle Geliştiricilerin Java tabanlı uygulamalar geliştirmesine yönelik olan Java Geliştirme Kitidir. JRE, Java programının çalıştığı Java Çalışma Zamanı Ortamıdır. JDK, JRE'yi ayrılmaz bir parçası olarak taşır. JRE ayrı ayrı kurulabileceği gibi herhangi bir geliştirme yapılmayacak, sadece Java tabanlı uygulamayı çalıştırmamız gereken veya Java programı çalıştırılacak sistemlere de kurulabilir. JVM bir spesifikasyondur ve spesifikasyonlara uyduğu sürece farklı uygulamalara sahip olabilir. Teknik özellikler docs.oracle.com bağlantısında bulunabilir. JRE, JVM'nin bir uygulamasıdır.

JDK

JDK, bir Java Programını derlemek, hata ayıklamak ve yürütmek için gereken tüm araçları, yürütülebilir dosyaları ve ikili dosyaları içeren Java Geliştirme Kitinin (Java Development Kit) kısaltmasıdır . JDK platforma bağlıdır, yani Windows, Mac ve Unix sistemleri için ayrı yükleyiciler vardır. JDK, hem JVM'yi hem de JRE'yi içerir ve kodun yürütülmesinden tamamen sorumludur. Java'nın bir sürümünü temsil eden JDK sürümüdür.

JRE

JRE, JVM'nin uygulanması olan bir Java Çalışma Zamanı Ortamıdır (Java Runtime Environment) yani JVM'de tanımlanan özellikler uygulanır ve kodun yürütülmesi için karşılık gelen bir ortam yaratır. JRE, fiziksel olarak var olan JVM gibi programı yürütmek için temel olarak Java ikili dosyalarını ve diğer sınıfları içerir. JRE, Java ikili dosyalarının yanı sıra çeşitli dağıtım teknolojilerinden, yürütülen kodla etkileşimde bulunmak için kullanıcı arayüzlerinden, farklı işlevler için bazı temel kitaplıklardan ve dil ve kullanım tabanlı kitaplıklardan oluşur .

JVM

JVM, Java bayt kodunun çalıştırılabileceği bir çalışma zamanı ortamı sağlayan bir spesifikasyon olan Java Sanal Makinesi (Java Virtual Machine) kısaltmasıdır yani soyut bir şeydir ve uygulanması algoritma seçiminden bağımsızdır ve Sun ve diğer şirketler tarafından sağlanmıştır. Bayt kodunu makineye özel koda dönüştürmekten sorumlu olan JVM'dir. Ayrıca başka dillerde yazılmış ve Java bayt koduna derlenmiş programları da çalıştırabilir. JVM belirtilen görevleri yerine getirir: Kodu yükler, Kodu doğrular, Kodu yürütür ve Çalışma zamanı ortamı sağlar.

JDK, JRE ve JVM arasındaki fark

JDK, JRE ve JVM arasındaki önemli farklar aşağıdadır.

1. Tanım (Definition) (JDK): JDK (Java Development Kit), Java'da uygulamalar geliştirmeye yönelik bir yazılım geliştirme kitidir. JRE'ye ek olarak JDK ayrıca çok sayıda geliştirme aracı (derleyiciler, JavaDoc, Java Hata Ayıklayıcı vb.) içerir.

1. Tanım (Definition) (JRE): JRE (Java Runtime Environment), JVM'nin uygulanmasıdır ve Java programlamayla yazılmış uygulamaları çalıştırmak için Java sınıfı kitaplıklarının yanı sıra Java Sanal Makinesi (JVM) ve diğer bileşenleri sağlayan bir yazılım paketi olarak tanımlanır.

1. Tanım (Definition) (JVM): JVM (Java Sanal Makinesi), platforma bağımlı olan ve spesifikasyon olarak üç kavrama sahip soyut bir makinedir; JVM uygulamasının gerekliliklerini açıklayan bir belge, uygulama, JVM gereksinimlerini karşılayan bir bilgisayar programı ve örnek olarak Java'yı çalıştıran bir uygulama bayt kodu, Java bayt kodunu yürütmek için bir çalışma zamanı ortamı sağlar.

2. Birinci sınıf işlevsellik (Prime functionality) (JDK): JDK öncelikle kod yürütme için kullanılır ve temel geliştirme işlevlerine sahiptir.

2. Birinci sınıf işlevsellik (Prime functionality) (JRE): Öte yandan JRE, esas olarak kod yürütme ortamının oluşturulmasından sorumludur.

2. Birinci sınıf işlevsellik (Prime functionality) (JVM): JVM ise tüm uygulamaları belirler ve bu uygulamaların JRE'ye sağlanmasından sorumludur.

3. Platform Bağımsızlığı (Platform Independence) (JDK): JDK platforma bağlıdır, yani farklı platformlar için farklı JDK gerekir.

3. Platform Bağımsızlığı (Platform Independence) (JRE): JDK gibi JRE de platforma bağlıdır.

3. Platform Bağımsızlığı (Platform Independence) (JVM): JVM platformdan bağımsızdır.

4. Aletler (Tools) (JDK): JDK ana geliştirmeden sorumlu olduğundan, Java uygulamasını geliştirmek, hata ayıklamak ve izlemek için araçlar içerir.

4. Aletler (Tools) (JRE): Öte yandan JRE, derleyici, hata ayıklayıcı vb. araçları içermez. Bunun yerine, JVM'nin programı çalıştırmak için ihtiyaç duyduğu sınıf kitaplıklarını ve diğer destekleyici dosyaları içerir.

4. Aletler (Tools) (JVM): JVM, yazılım geliştirme araçlarını içermez.

5. Uygulama (Implementation) (JDK): JDK = Java Çalışma Zamanı Ortamı (JRE) + Geliştirme araçları

5. Uygulama (Implementation) (JRE): JRE = Java Sanal Makinesi (JVM) + Uygulamayı çalıştırmak için Kitaplıklar

5. Uygulama (Implementation) (JVM): JVM = Java bayt kodunu yürütmek için yalnızca Çalışma Zamanı ortamı.
